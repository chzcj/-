# 育见官方网站

Anthropic 编辑排版 × Kimi 产品舞台，育见绿 `#6f9f56` 作强调色。

## 预览

```bash
cd website && python3 -m http.server 8080
```

浏览器打开 `http://localhost:8080`。

## 设计要点

- 底色：`#faf9f5` ivory + `#f5f5f7` 灰带交替，无突兀跳色
- 截图：`#f8f6e5` 设备框 + 1px hairline + 轻阴影，边界清晰
- 布局：Hero / 产品区 / 技术区均为均衡双栏或居中网格
- 产品区：滚动驱动 sticky 展示 + 首次相识纵向交替叙事

## 渠道跳转（小程序 / 公众号 / 自媒体）

### 实现方式

| 场景 | 行为 |
|------|------|
| PC / 手机外部浏览器 | 点击「开启 AI 陪伴」或生态矩阵二维码卡片 → 弹窗展示高清码 |
| 微信内置浏览器 | 若已配置加密跳转链接 → 一键进入育见；否则打开 Web 版 |
| 抖音 / 知乎 / B站 | 页脚生态矩阵外链，新标签页打开主页（需在配置中填写 URL） |

### 配置

编辑 `js/channels.js`：

1. **育见 AI**：替换 `assets/qr/miniprogram.svg` 为正式码；可选填写 `urlLink`（微信后台生成的 `https://wxaurl.cn/xxx`）
2. **公众号**：替换 `assets/qr/wechat-oa.svg`；可选填写 `articleUrl`（图文永久链接，微信内直达）
3. **视频号**：替换 `assets/qr/channels-video.svg`
4. **自媒体外链**：填写 `douyin` / `zhihu` / `bilibili` 等 `url` 字段，未填写的入口自动隐藏

素材说明见 `assets/qr/README.md`。

### 入口位置

- 导航栏固定按钮「开启 AI 陪伴」
- Hero / 合作区 / 页脚产品栏
- 产品区「扫码开启 AI 成长陪伴系统」
- 页脚「生态矩阵」板块（小程序 · 公众号 · 视频号 · 自媒体）

## 修改后自检（必做）

每次改动官网页面或前端代码后，逐项确认：

### 1. 跳转兼容性

- [ ] **微信内置浏览器**：「开启 AI 陪伴」可一键进入（已配 urlLink）或正常打开 Web 版
- [ ] **PC / 手机自带浏览器**：点击入口弹出二维码弹窗，无空白或失效按钮
- [ ] **自媒体外链**：已配置的平台可新标签打开；未配置项已从页脚隐藏

### 2. 文案去技术术语

对外文案禁止出现：`小程序`、`web端`、`链接 scheme`、`url link` 等开发词汇。

| 避免 | 改用 |
|------|------|
| 点击跳转小程序 | 扫码开启 AI 成长陪伴系统 |
| Web 版入口 | 开启 AI 陪伴 |

### 3. 视觉规范

- [ ] 所有二维码无边框，低饱和融合页脚底色
- [ ] 弹窗简洁，标题 + 一句说明 + 居中二维码
- [ ] 生态矩阵与全站 Anthropic 风格一致，不突兀割裂

### 4. 链路逻辑一致性

| 入口 | 业务定位 |
|------|----------|
| 育见 AI | 心智陪伴系统 |
| 公众号 / 知乎 | 一合学社 · 家庭教育文库 |
| 视频号 / 抖音 | 九州问学 · 支教纪实与教育纪录片 |

## 落地优先级

1. **已上线**：页脚二维码弹窗 + 生态矩阵（零后端、全设备兼容）
2. **迭代**：微信内一键跳转（配置 urlLink + 可选公众号 JS-SDK）
3. **同步**：自媒体主页外链填入 `channels.js` 后自动展示
