# 各平台上线发布指南

> **⚠️ 免责声明**：本文档中的审核规则、资质要求等基于编写时的各平台政策整理，可能随平台政策调整而变化。实际上线前请以各平台最新官方政策为准。如发现过时信息，请及时更新本文档或查阅官方文档确认。

## 通用上线流程

```
注册账号 → 资质认证 → 开发完成 → 代码上传 → 提交审核 → 审核通过 → 发布上线
```

## 微信小程序

### 账号注册
1. 访问 https://mp.weixin.qq.com/
2. 点击"立即注册"，选择"小程序"
3. 填写邮箱、密码等信息
4. 邮箱验证后完善主体信息

### 资质认证
- 个人主体：无需认证，但功能受限（无支付等）
- 企业主体：需提交营业执照，认证费 300 元/年
- 认证后可开通支付、直播等高级能力

### 代码上传
1. 在微信开发者工具中点击"上传"
2. 填写版本号和项目备注
3. 登录 mp.weixin.qq.com → 版本管理 → 开发版本中查看

### 审核要点
- 类目选择必须与实际功能一致
- 涉及支付需先开通微信支付商户号
- 涉及用户信息需有隐私协议
- 服务类目资质要求（如餐饮需食品经营许可证）
- 不得诱导分享、诱导关注

### 常见驳回原因
1. 功能不完整或无法使用
2. 类目与功能不符
3. 缺少必要资质
4. 存在诱导行为
5. 内容违规

## 支付宝小程序

### 账号注册
1. 访问 https://open.alipay.com/
2. 使用企业支付宝账号登录
3. 进入开发者中心创建小程序

### 资质认证
- 必须为企业主体（不支持个人）
- 需完成企业实名认证
- 部分行业需额外资质

### 审核要点
- 功能必须可用且完整
- 不得有虚拟支付（iOS 限制）
- 涉及资金需接入支付宝资金安全方案
- 隐私协议必须完善

## 灰度发布策略

审核通过后，建议采用灰度发布：

1. **按比例灰度**：先发布给 1% → 5% → 20% → 100% 的用户
2. **按地区灰度**：先在小范围地区发布，观察稳定性
3. **回滚机制**：发现问题及时回退到上一版本

微信小程序灰度操作路径：mp.weixin.qq.com → 版本管理 → 线上版本 → 灰度发布

## 版本管理建议

- 使用语义化版本号：`主版本.次版本.修订号`
- 每次上传附带清晰的版本说明
- 保留最近 3 个稳定版本的代码分支
- 重大更新前先在体验版充分测试

## 上线前检查清单

- [ ] 核心功能在真机上测试通过
- [ ] 所有页面加载正常，无白屏
- [ ] 登录流程完整可用
- [ ] 支付流程测试通过（如有）
- [ ] 分享功能正常
- [ ] 隐私协议已添加
- [ ] 类目和资质已准备就绪
- [ ] 无敏感信息硬编码（API Key 等）
- [ ] 包大小符合平台限制
- [ ] 性能指标达标（首屏 < 2s）
