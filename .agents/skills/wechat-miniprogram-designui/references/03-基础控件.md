# 三、基础控件规范

## 3.1 按钮 Button

- **组件/场景**：主按钮、次按钮、文字按钮、危险按钮、禁用按钮
- **适用场景**：所有页面操作、提交、确认、跳转、取消操作
- **官方标准规范**：按钮分为主要按钮（Primary）、次要按钮（Default）、危险按钮（Warn）、文字按钮四种基础类型，层级区分明确。WeUI 中对应 `weui-btn_primary`、`weui-btn_default`、`weui-btn_warn`。
- **尺寸/参数**：
  - 通栏按钮高度：**88rpx**
  - 小型按钮高度：**64rpx**
  - 按钮圆角：**8rpx**
  - 按钮文字：**28rpx**，字重 500
  - 按钮左右内边距：**32rpx**
  - 弹窗底部双按钮间距：**16rpx**
- **颜色规范**：

  | 类型 | 背景色 | 文字色 | 边框 |
  |---|---|---|---|
  | 主按钮 (Primary) | `#07C160` | `#FFFFFF` | 无 |
  | 主按钮按下态 | `#06AD56` | `#FFFFFF` | 无 |
  | 次按钮 (Default) | `#FFFFFF` | `rgba(0,0,0,0.9)` | `rgba(0,0,0,0.2)` |
  | 次按钮按下态 | `rgba(0,0,0,0.05)` | `rgba(0,0,0,0.9)` | `rgba(0,0,0,0.2)` |
  | 危险按钮 (Warn) | `#FA5151` | `#FFFFFF` | 无 |
  | 危险按钮按下态 | `#E04848` | `#FFFFFF` | 无 |
  | 文字按钮 | 透明 | `#576B95` | 无 |
  | 禁用按钮 | `rgba(0,0,0,0.05)` | `rgba(0,0,0,0.2)` | 无 |

- **交互规则**：点击有视觉反馈（背景色加深约 6–10%），禁用状态无点击效果；弹窗底部按钮默认左取消（次按钮）、右确认（主按钮）。
- **WeUI 对应类名**：
  - 主按钮：`weui-btn weui-btn_primary`
  - 次按钮：`weui-btn weui-btn_default`
  - 危险按钮：`weui-btn weui-btn_warn`
  - 小型按钮：`weui-btn weui-btn_mini`
  - 禁用态：`weui-btn_disabled`
- **强制要求**：
  - 关键操作必须使用主按钮
  - 取消、返回次要操作使用次按钮 / 文字按钮
  - 单页仅保留**一个核心主按钮**
- **禁止设计**：
  - 主按钮过多、页面多个高亮主按钮并存
  - 按钮尺寸过小导致误触（高度 < 64rpx）
  - 禁用状态视觉不明显（与正常态接近）
  - 危险按钮用于非危险操作（如"确定"用红色）
- **最佳实践**：单页面仅保留**一个核心主按钮**，弱化次要操作，减少用户选择成本。主要操作用品牌绿主按钮，取消/返回用白色次按钮，删除/注销用红色危险按钮。

## 3.2 输入框 Input

- **组件/场景**：表单输入、搜索输入、信息填写输入框
- **适用场景**：登录、表单、搜索、资料编辑页面
- **官方标准规范**：输入框需展示清晰边框、占位提示文字、输入状态区分，支持输入校验反馈。WeUI 中对应 `weui-input`。
- **尺寸/参数**：
  - 输入框高度：**80rpx**
  - 内边距左右：**20rpx**
  - 文字：**28rpx**
  - 占位符文字：**28rpx**，颜色 `rgba(0,0,0,0.2)`
- **状态颜色**：

  | 状态 | 边框色 | 说明 |
  |---|---|---|
  | 默认 | `rgba(0,0,0,0.1)` | 常规未聚焦 |
  | 聚焦 (Focus) | `#07C160` | 品牌绿高亮 |
  | 错误 (Error) | `#FA5151` | 红色边框 + 错误提示文字 |
  | 禁用 | `rgba(0,0,0,0.05)` 背景 | 不可编辑 |

- **交互规则**：聚焦高亮（品牌绿边框）、失焦恢复默认；输入错误时边框变红并展示文字提示（`#FA5151`，24rpx 辅助文字）；支持清空按钮（右侧 x 图标）。
- **WeUI 对应类名**：`weui-input`（输入框本体）、`weui-cell__ft`（右侧附加区，如清空按钮）
- **强制要求**：
  - 必填项需标注（红色星号 `*` 或文字提示）
  - 占位文案清晰说明输入规则
  - 校验失败需给出明确错误提示
- **禁止设计**：
  - 输入框无边框难以识别
  - 无错误提示
  - 输入区域过小
  - 占位符颜色过深导致与输入内容混淆
- **最佳实践**：搜索框默认圆角 **16rpx**，表单输入框圆角 **8rpx**，区分场景样式。输入框搭配标签（label）使用，标签宽度统一。

## 3.3 单选 / 多选 / 开关

- **组件/场景**：单选框（Radio）、复选框（Checkbox）、开关（Switch）
- **适用场景**：选项选择、协议勾选、状态开启关闭
- **官方标准规范**：选择控件状态区分清晰，默认状态、选中状态、禁用状态视觉差异化。WeUI 中对应 `weui-check`、`weui-switch`。
- **尺寸/参数**：
  - 单选 / 复选框尺寸：**40rpx**（20px @2x）
  - 搭配文字：**28rpx**
  - 整行点击热区：≥ **88rpx** 高度
  - 选项间距：**24rpx**
- **颜色规范**：

  | 控件 | 选中色 | 未选中色 |
  |---|---|---|
  | 单选框 | `#07C160`（填充） | `rgba(0,0,0,0.2)`（描边） |
  | 复选框 | `#07C160`（填充 + 对勾） | `rgba(0,0,0,0.2)`（描边） |
  | 开关开启 | `#07C160`（背景） | — |
  | 开关关闭 | `rgba(0,0,0,0.2)`（背景） | — |

- **开关尺寸**：高度 **52rpx**（26px @2x），宽度约为高度的 1.6 倍
- **交互规则**：点击控件 + 文字区域均可触发选择（扩大热区）；开关切换有滑动动效（transition 约 0.2s）。
- **强制要求**：协议勾选必须为用户主动勾选，**禁止默认勾选**（审核红线）。
- **禁止设计**：
  - 选择状态模糊（选中/未选中区分不明显）
  - 点击区域过小（整行热区 < 88rpx）
  - 隐私协议默认勾选 ⚠️（审核红线）
  - 开关用于需二次确认的危险操作
- **最佳实践**：重要协议、授权选项，留白充足，引导用户主动操作。单选项少于 5 个时用单选，多于 5 个时考虑用 Select/Picker 组件。

## 3.4 搜索框 SearchBar

- **组件/场景**：搜索输入框
- **适用场景**：页面顶部搜索、列表筛选搜索
- **尺寸/参数**：
  - 圆角：**16rpx**
  - 高度：**64–72rpx**
  - 左右内边距：**24rpx**
  - 搜索图标：**32rpx**，颜色 `rgba(0,0,0,0.4)`
  - 占位文字：**28rpx**，颜色 `rgba(0,0,0,0.2)`
- **强制要求**：搜索框需与页面其他内容有明确视觉分隔，底色建议 `#EDEDED`。
- **交互规则**：聚焦后弹出搜索结果列表或历史搜索；支持右侧取消按钮（`#576B95` 文字按钮）。
