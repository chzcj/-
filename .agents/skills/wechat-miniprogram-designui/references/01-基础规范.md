# 一、设计基础规范

## 1.1 布局、安全区与适配规范

- **组件/场景**：小程序全局布局、屏幕适配、安全区、胶囊按钮避让
- **适用场景**：所有小程序页面整体布局、页面适配、异形屏适配
- **官方标准规范**：小程序需适配全尺寸手机屏幕，针对异形屏、全面屏预留系统安全区域，规避微信原生胶囊（更多、关闭）控件，页面内容不可被系统控件遮挡；支持深色/浅色模式自适应展示。
- **尺寸/参数**：
  - 页面基础单位 `rpx`；`750rpx = 设备全屏宽度`
  - 顶部系统胶囊高度区间：**88rpx**
  - 顶部安全区高度：普通手机 0rpx，异形屏 **44rpx**
  - 底部安全区高度：异形屏 **34rpx**
- **交互规则**：所有可交互控件需远离系统原生控件，避免点击冲突；页面上下留白适配安全区，无内容裁切、无控件遮挡。
- **强制要求**：核心内容、按钮、输入框、关键文案必须全部落在安全区内；所有页面必须兼容异形全面屏。
- **禁止设计**：
  - 页面内容紧贴屏幕边缘
  - 可点击控件覆盖微信原生胶囊
  - 页面内容被系统状态栏、底部小黑条裁切
- **最佳实践**：全局开启安全区适配，页面顶部预留 100rpx 避让胶囊，底部 tab 页面预留安全边距，非 tab 页面根据机型自适应底部留白。

## 1.2 栅格布局规范

- **组件/场景**：页面栅格、间距布局、整体版式规范
- **适用场景**：所有小程序页面排版、列表、卡片、模块布局
- **官方标准规范**：采用统一 8px 栅格体系（小程序中 8px = 16rpx），以 16rpx 为最小间距单位，统一页面层级，区分内外边距层级。
- **尺寸/参数**：
  - 基础间距梯度（基于 8px 栅格）：**16rpx、24rpx、32rpx、48rpx**
  - 最小间距单位：**16rpx**（8px）
  - 模块内边距：**32rpx**（16px）
  - 模块外边距：**24rpx**（12px）
  - 卡片内边距：**32rpx**
- **交互规则**：同页面间距统一，层级模块间距区分明显，视觉层级清晰。
- **强制要求**：页面模块间距统一规范，基于 8px 栅格，无杂乱间距混用。
- **禁止设计**：
  - 同一页面多种无规律间距
  - 间距跳离 8px 栅格体系
  - 模块紧贴屏幕边缘无留白
- **最佳实践**：移动端小程序优先使用 16rpx/24rpx/32rpx 基础间距，复杂页面统一全局间距规则，保证视觉一致性。

## 1.3 字体规范

- **组件/场景**：全局字体、文案字号、行高、字重、字体族
- **适用场景**：小程序所有文案、标题、正文、辅助文字、备注文字
- **官方标准规范**：小程序默认跟随系统字体，统一层级字体体系，区分标题、正文、辅助文本层级，保证可读性。
- **尺寸/参数**：

  | 层级 | 字号 | 行高 | 字重 |
  |---|---|---|---|
  | 大标题 | 36rpx | 48rpx | Bold (700) |
  | 页面标题 | 32rpx | 44rpx | Bold (700) |
  | 正文常规 | 28rpx | 40rpx | Regular (400) |
  | 辅助文字 | 24rpx | 32rpx | Regular (400) |

- **系统字体栈**：
  ```
  -apple-system, BlinkMacSystemFont, "PingFang SC", "Helvetica Neue", "Microsoft YaHei", sans-serif
  ```
  - iOS 优先使用 SF Pro（英文/数字）+ PingFang SC（中文）
  - Android 优先使用 Roboto（英文/数字）+ Microsoft YaHei / Noto Sans CJK（中文）
- **交互规则**：纯展示文字不可被点击误触，可点击文字需自带点击反馈（颜色变化或下划线）。
- **强制要求**：最小可读文字不低于 **24rpx**，保证移动端可读性。不使用非系统字体（除非通过 `wx.loadFontFace` 加载并有明确设计意图）。
- **禁止设计**：
  - 正文文字小于 24rpx
  - 页面字体层级混乱、字号随意混用
  - 使用未经加载的自定义字体导致回退不可控
- **最佳实践**：全局统一字体层级，标题加粗、常规正文常规字重、辅助文字浅色系弱化展示。

## 1.4 颜色体系（完整官方色值）

- **组件/场景**：全局色彩、品牌色、功能色、文字色阶、背景色、分割线
- **适用场景**：所有组件、卡片、弹窗、按钮、文字、背景视觉样式
- **官方标准规范**：色彩区分品牌色、功能色（成功/警示/提示/信息）、四级文字色阶、背景色、分割线色。所有功能色全项目统一，不得自定义。

### 品牌色

| 用途 | 色值 | 说明 |
|---|---|---|
| 微信品牌绿 | `#07C160` | 小程序主色调，核心按钮、选中态、开关开启态 |
| 微信深绿 | `#1AAD19` | 辅助绿色，部分场景替代品牌绿 |

### 功能色（四个标准功能色）

| 功能 | 色值 | 使用场景 |
|---|---|---|
| 成功（Success） | `#07C160` | 操作成功、完成状态、成功 Toast |
| 警示（Error/Danger） | `#FA5151` | 错误提示、删除操作、危险按钮、失败 Toast |
| 提示（Warning） | `#FFC300` | 警告提示、注意事项、待处理状态 |
| 信息（Info） | `#10AEFF` | 信息说明、帮助提示、普通通知 |

### 文字色阶（四级）

| 层级 | 色值 | 用途 |
|---|---|---|
| 一级文字 | `rgba(0, 0, 0, 0.9)` | 页面标题、列表标题、正文核心内容 |
| 二级文字 | `rgba(0, 0, 0, 0.6)` | 列表摘要、次要说明文字 |
| 三级文字 | `rgba(0, 0, 0, 0.4)` | 辅助信息、时间戳、来源标注 |
| 四级文字 | `rgba(0, 0, 0, 0.2)` | 输入框占位符、禁用态文字 |

> **为什么用 rgba 而非固定 hex？** rgba 写法在明/暗模式下能自动适配不同背景，推荐小程序中优先使用 rgba 定义文字色。

### 背景色

| 用途 | 色值 |
|---|---|
| 页面背景（Page Background） | `#F6F6F6` |
| 白色背景（卡片/列表/弹窗/导航栏） | `#FFFFFF` |

### 分割线与边框

| 用途 | 色值 |
|---|---|
| 列表分割线 | `rgba(0, 0, 0, 0.1)` |
| 卡片边框 | `rgba(0, 0, 0, 0.1)` |
| 输入框边框（默认） | `rgba(0, 0, 0, 0.1)` |
| 输入框边框（聚焦） | `#07C160` |
| 输入框边框（错误） | `#FA5151` |

### 遮罩与链接

| 用途 | 色值 |
|---|---|
| 弹窗遮罩层（Modal Overlay） | `rgba(0, 0, 0, 0.6)` |
| 文字链接色 | `#576B95` |
| 链接按下态 | `#4A6F8A` |

### 按钮颜色

| 类型 | 背景色 | 文字色 |
|---|---|---|
| 主按钮（Primary） | `#07C160` | `#FFFFFF` |
| 主按钮按下态 | `#06AD56` | `#FFFFFF` |
| 次按钮（Default） | `#FFFFFF` | `rgba(0, 0, 0, 0.9)` |
| 次按钮边框 | `rgba(0, 0, 0, 0.2)` | — |
| 危险按钮（Warn） | `#FA5151` | `#FFFFFF` |
| 禁用按钮 | `rgba(0, 0, 0, 0.05)` | `rgba(0, 0, 0, 0.2)` |

- **交互规则**：不同状态（默认、按下、禁用）色彩区分明显。按钮按下态背景变深约 6–10%。
- **强制要求**：功能色统一（成功、失败、警告、提示）全项目统一，不得自定义功能色。
- **禁止设计**：
  - 主色用于非核心操作按钮
  - 自定义功能色替代标准成功/警示/提示色
  - 文字色阶跳过层级（如从一级直接跳到四级）
- **最佳实践**：
  - 主色（品牌绿 `#07C160`）仅用于主按钮、选中态、开关
  - 链接色（`#576B95`）用于文字链接、可点击文本
  - 功能色各司其职：红=错、绿=对、黄=警、蓝=信息
  - 文字色用 rgba 而不用固定 hex，方便暗黑模式适配

## 1.5 暗黑模式（Dark Mode）规范

- **组件/场景**：全局暗黑模式适配
- **适用场景**：所有小程序页面
- **官方标准规范**：小程序支持系统级暗黑模式（iOS 13+ / Android 10+），通过 `@media (prefers-color-scheme: dark)` 或 `wx.getSystemInfoSync().theme` 检测并切换。`app.json` 中 `darkmode: true` 开启支持。
- **强制要求**：
  - `app.json` 配置 `"darkmode": true`
  - 在 `theme.json` 中定义明暗主题变量
  - 页面/组件使用 CSS 变量引用主题色，不可写死色值
  - 图片资源需准备明暗两套（或使用矢量图标自适应）

### 暗黑模式颜色映射

| 用途 | 浅色模式 | 暗黑模式 |
|---|---|---|
| 页面背景 | `#F6F6F6` | `#191919` |
| 卡片/列表背景 | `#FFFFFF` | `#1F1F1F` |
| 一级文字 | `rgba(0,0,0,0.9)` | `rgba(255,255,255,0.9)` |
| 二级文字 | `rgba(0,0,0,0.6)` | `rgba(255,255,255,0.6)` |
| 三级文字 | `rgba(0,0,0,0.4)` | `rgba(255,255,255,0.4)` |
| 四级文字 | `rgba(0,0,0,0.2)` | `rgba(255,255,255,0.2)` |
| 分割线 | `rgba(0,0,0,0.1)` | `rgba(255,255,255,0.1)` |
| 品牌绿 | `#07C160` | `#07C160`（不变） |
| 链接色 | `#576B95` | `#788FAA` |
| 遮罩层 | `rgba(0,0,0,0.6)` | `rgba(0,0,0,0.8)` |

### CSS 变量声明示例

```css
/* 浅色模式（默认） */
page {
  --bg-page: #F6F6F6;
  --bg-card: #FFFFFF;
  --text-primary: rgba(0, 0, 0, 0.9);
  --text-secondary: rgba(0, 0, 0, 0.6);
  --text-tertiary: rgba(0, 0, 0, 0.4);
  --divider: rgba(0, 0, 0, 0.1);
}

/* 暗黑模式 */
@media (prefers-color-scheme: dark) {
  page {
    --bg-page: #191919;
    --bg-card: #1F1F1F;
    --text-primary: rgba(255, 255, 255, 0.9);
    --text-secondary: rgba(255, 255, 255, 0.6);
    --text-tertiary: rgba(255, 255, 255, 0.4);
    --divider: rgba(255, 255, 255, 0.1);
  }
}
```

- **禁止设计**：
  - 写死色值而不使用 CSS 变量
  - 暗黑模式下保持纯白卡片背景
  - 图片无暗黑适配（白色背景图片在暗黑模式显突兀）
  - 品牌绿在暗黑模式下调亮（官方要求品牌绿不变）
- **最佳实践**：
  - 全部颜色通过 CSS 变量引用，一处理论切换
  - 文字色用 rgba 天然适配明暗切换
  - 图标/插图准备明暗两套，或用单色矢量图标
  - Tabbar 图标准备 `_dark` 后缀的暗色版本

## 1.6 圆角与阴影规范

- **组件/场景**：圆角、阴影视觉基础规范
- **适用场景**：所有组件、卡片、弹窗、按钮视觉样式
- **尺寸/参数**：
  - 标准圆角梯度：**8rpx、12rpx、16rpx**
  - 卡片默认 12rpx 圆角
  - 按钮默认 8rpx 圆角
  - 搜索框默认 16rpx 圆角
  - 弹窗默认 16rpx 圆角
  - 头像/图标等圆形元素：`border-radius: 50%`
- **阴影**：
  - 卡片阴影：轻量，`0 2rpx 12rpx rgba(0,0,0,0.08)`
  - 弹窗阴影：`0 8rpx 40rpx rgba(0,0,0,0.12)`
  - 禁止使用厚重阴影、彩色阴影、霓虹效果
- **强制要求**：同类型组件统一圆角值。
- **禁止设计**：
  - 过度圆角（> 24rpx 非特殊场景）
  - 尖锐直角与圆角混用无规律
  - 厚重阴影、彩色阴影
- **最佳实践**：全局统一三个圆角梯度，卡片/弹窗/按钮各司其值，确保页面整洁统一。
