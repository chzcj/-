# 二、导航类组件规范

## 2.1 顶部导航栏（Navbar）

- **组件/场景**：小程序页面顶部原生导航栏、自定义导航栏
- **适用场景**：所有小程序页面顶部导航
- **官方标准规范**：分为原生导航栏与自定义导航栏，原生导航栏由微信系统控制，自定义导航栏需严格避让系统胶囊控件。
- **尺寸/参数**：
  - 原生导航栏高度：**88rpx**
  - 标题文字：**32rpx**，字重 Bold
  - 导航栏文字居中展示
  - 自定义导航栏总高度（含状态栏）：iOS 异形屏 ~**132rpx**（44rpx 状态栏 + 88rpx 导航栏）
- **交互规则**：原生自带返回按钮；自定义导航栏需自带返回功能，且点击区域充足（≥ 88rpx）。
- **强制要求**：
  - 自定义导航栏必须避让右上角系统胶囊（胶囊区域不可放置任何交互元素）
  - 必须配备返回入口（返回按钮或左滑返回）
  - 标题文字不可过长导致与胶囊重叠
- **禁止设计**：
  - 自定义导航栏遮挡系统胶囊按钮
  - 去除返回功能
  - 导航栏文字溢出截断无处理
  - 在胶囊正下方放置可点击元素（容易误触）
- **最佳实践**：通用页面优先使用原生导航栏，仅个性化页面使用自定义导航，保证兼容性。自定义导航栏内容在胶囊左侧区域布局。

## 2.2 底部 Tabbar

- **组件/场景**：小程序底部标签导航栏
- **适用场景**：多页面常驻首页、分类、我的等核心页面
- **官方标准规范**：Tabbar 为全局常驻导航，用于区分小程序核心一级页面，数量严格限制。
- **尺寸/参数**：
  - Tabbar 内容高度：**98rpx**
  - 异形屏底部安全区：**34rpx**，需额外计入容器总高度
  - 异形屏 Tabbar 容器总高度建议：**132rpx**（98rpx 内容区 + 34rpx 安全区）
  - 图标尺寸：**48rpx**（24px @2x）
  - Tab 图标设计稿标准：**80rpx × 80rpx**（40px × 40px @2x），视觉区域建议控制在 48rpx 以内
  - 文字：**20rpx**
  - Tab 数量：**2–5 个**
- **交互规则**：点击切换页面，选中态高亮，未选中态弱化；支持页面常驻不刷新；切换无过渡动画（原生 Tabbar 为即时切换）。
- **强制要求**：
  - Tab 数量必须 2–5 个（1 个或 6+ 个直接审核不通过）
  - 选中、未选中状态区分清晰（颜色 + 图标双切换）
  - Tabbar 背景必须覆盖内容区和底部安全区，图标、文字必须完整落在背景内
  - 自定义 Tabbar 也遵循相同数值和数量限制
- **禁止设计**：
  - Tab 数量 1 个或大于 5 个
  - Tab 无文字仅图标
  - Tab 无图标仅文字
  - Tab 状态无区分
  - 动态随意隐藏显示 Tabbar
  - 仅设置 98rpx 总高度却再叠加 34rpx 底部安全区，导致图标/文字被挤出背景或贴近底部小黑条
  - 图标尺寸不标准、模糊、比例失调
- **最佳实践**：核心功能放置 Tab 页面，次要功能放入页面内跳转，保证用户认知统一；实现时将 Tabbar 拆为 98rpx 内容区 + 34rpx 底部安全区，背景通栏覆盖总高度，内容垂直居中在 98rpx 区域内。

### Tabbar 图标设计要点

- 选中/未选中各一套图标，建议用纯色填充图标
- 选中态图标使用品牌绿 `#07C160`
- 未选中态图标使用 `rgba(0, 0, 0, 0.4)` 或 `#888888`
- 图标绘制在 80rpx × 80rpx 画布内，视觉内容在中央 48rpx × 48rpx 区域
- 暗黑模式需准备 `_dark` 后缀图标（或使用矢量 SVG 自适应）

## 2.3 顶部标签 Tab（页面内 Tab）

- **组件/场景**：页面内顶部分类 Tab 切换
- **适用场景**：同一页面多分类、多列表切换场景
- **官方标准规范**：页面内 Tab 用于同一页面内内容切换，不可作为页面级导航（不得 Tab 切换时 navigateTo 新页面）。
- **尺寸/参数**：
  - Tab 高度：**80rpx**
  - 文字：**28rpx**
  - 下划线宽度跟随文字宽度（或固定 48rpx）
  - 下划线高度：**4rpx**
- **交互规则**：点击切换内容，自带高亮下划线（品牌绿 `#07C160`），支持左右滑动切换（swiper 联动）。
- **强制要求**：默认选中首个 Tab，选中态直观可见（文字加粗 + 下划线）。
- **禁止设计**：
  - Tab 数量过多横向挤压（> 5 个）
  - 无选中状态
  - Tab 切换跳转新页面（应用 swiper 在当前页切换）
  - 下划线颜色使用非品牌色
- **最佳实践**：分类数量 ≤ 5 个，超出使用横向滚动 Tab（`scroll-view` + `enable-flex`）。

## 2.4 页面栈深度限制（导航硬性约束）

- **组件/场景**：页面跳转路由栈
- **适用场景**：所有涉页面间跳转的交互流程
- **官方标准规范**：小程序页面栈最大深度为 **10 层**（以 `wx.navigateTo` 计数）。
- **尺寸/参数**：无
- **交互规则**：
  - `wx.navigateTo` 每次调用栈深度 +1，最多 10 次
  - 超过 10 层后 `navigateTo` 静默失败，必须改用 `wx.redirectTo`（替换当前页）或 `wx.reLaunch`（重启）
  - `wx.navigateBack` 返回上一页（delta ≤ 当前栈深度）
  - `wx.switchTab` 跳转 Tab 页，关闭所有非 Tab 页面
- **强制要求**：设计页面流程时，核心用户路径（如首页 → 列表 → 详情 → 下单 → 支付）不得超过 10 层。超过时必须用 `redirectTo` 断链。
- **禁止设计**：
  - 设计无限制深层跳转的用户流程
  - 列表 → 详情 → 推荐详情 → 更多详情 → … 的无限钻取
- **最佳实践**：
  - 同类型页面间用 `redirectTo` 替换而非 `navigateTo` 叠加（如详情 A → 详情 B）
  - 关键漏斗节点（如确认订单 → 支付成功）用 `redirectTo` 切断回退
  - 在设计稿的交互流程图中标注页面栈层级
