# 七、合规 & 审核红线（高危必看）⚠️

> 本章是审核驳回率最高的雷区。回答任何与隐私、授权、弹窗、仿系统 UI 相关的问题，**必须先对照本章红线**。

## 7.1 授权与隐私弹窗规范

- **组件/场景**：隐私授权、位置授权、手机号授权弹窗
- **适用场景**：所有需要获取用户权限、个人信息的小程序
- **官方标准规范**：所有权限获取必须用户主动触发，禁止启动强制弹窗、刷屏弹窗。隐私协议须在 `app.json` 中配置 `__usePrivacyCheck__: true`。
- **尺寸/参数**：遵循官方弹窗标准样式
- **交互规则**：用户点击对应功能后，方可唤起授权弹窗。
- **强制要求**：
  - 禁止小程序启动自动弹出隐私授权
  - 必须提供关闭、拒绝选项
  - 隐私协议可随时查看（通常在「我的」页面提供入口）
  - 用户拒绝授权后功能可降级使用，不可强制关闭小程序
- **禁止设计**：
  - 开机强制弹窗、无法关闭弹窗
  - 诱导授权（如虚假功能引导："点击领取"实际为授权）
  - 默认同意隐私协议
  - 拒绝授权后无法使用任何功能（应提供降级方案）
  - 未声明隐私接口用途就调用（需在 mp 后台「用户隐私保护指引」中声明）
- **最佳实践**：用户触发对应功能后按需授权，弹窗文案通俗易懂，明确告知授权用途。在「设置」页提供授权管理入口。

## 7.2 UI 审核通用禁忌

- **组件/场景**：全小程序 UI 合规审核
- **适用场景**：所有小程序上线审核
- **官方标准规范**：禁止模仿系统通知、系统弹窗、系统状态栏样式；禁止虚假警示 UI；禁止诱导点击。
- **交互规则**：所有弹窗、提示均可手动关闭，无强制阻塞。
- **强制要求**：所有自定义 UI 不可伪装系统官方提示。
- **禁止设计**：
  - 仿微信系统弹窗、仿手机系统通知
  - 虚假红包、虚假消息提示
  - 无法关闭广告弹窗
  - 仿系统状态栏（如显示假的"信号满格""电量 100%"）
  - 虚假的"红包来了""转账通知"等消息提醒
- **最佳实践**：所有自定义弹窗标注小程序自有样式，区分系统原生 UI。使用品牌色而非系统绿做弹窗按钮。

## 7.3 页面栈与跳转限制

- **组件/场景**：页面跳转、路由栈深度
- **适用场景**：多页面串联流程
- **官方标准规范**：
  - `wx.navigateTo` 页面栈最大深度 **10 层**
  - 超过 10 层后跳转静默失败，用户点不动
  - 开发工具中会触发 `wx.onPageNotFound` 或静默失败
- **交互规则**：
  - 超过 10 层时使用 `wx.redirectTo`（替换当前页，栈深度不变）
  - 跳转 Tab 页使用 `wx.switchTab`（关闭所有非 Tab 页）
  - 返回首页或重启使用 `wx.reLaunch`
- **强制要求**：核心流程路径深度 ≤ 10 层。同类页面间跳转优先用 `redirectTo`。
- **禁止设计**：无限深钻的浏览路径（如：列表 → 详情A → 推荐详情B → 推荐详情C → …）
- **最佳实践**："返回首页"按钮用 `reLaunch`，详情页之间的"相关推荐"跳转用 `redirectTo` 替换当前页。

## 八、禁忌设计 & 反面案例汇总（上线前自查清单）

- **组件/场景**：全场景小程序设计避雷
- **适用场景**：所有小程序设计、自查、上线验收
- **官方标准规范**：所有设计需遵循微信开放平台 UI 规范，保证用户体验、避免误导用户。
- **交互规则**：无强制阻塞用户操作、无诱导操作交互。
- **强制要求**：
  - 不得篡改、遮挡微信官方原生控件（胶囊按钮、状态栏、底部安全区）
  - 不得仿系统 UI
  - 不得强制用户操作

### 必查禁忌清单（逐项打勾）

| # | 禁忌项 | 风险等级 | 后果 |
|---|---|---|---|
| 1 | 遮挡胶囊按钮、状态栏、底部安全区 | 🔴 高危 | 审核驳回 |
| 2 | 无限弹窗、无法关闭弹窗 | 🔴 高危 | 审核驳回 |
| 3 | 默认勾选隐私、用户协议 | 🔴 高危 | 合规违规，强制整改 |
| 4 | 仿系统通知、红包、消息弹窗诱导点击 | 🔴 高危 | 审核驳回 + 可能封禁 |
| 5 | 启动自动弹隐私/授权弹窗 | 🔴 高危 | 审核驳回 |
| 6 | Tabbar 数量超出 2–5 个范围 | 🔴 高危 | 审核驳回（技术上也无效） |
| 7 | 页面多主按钮、视觉层级混乱 | 🟡 中等 | 体验差，可能被拒 |
| 8 | 文字过小（< 24rpx）、排版杂乱、间距不统一 | 🟡 中等 | 体验差，可能被拒 |
| 9 | 无状态反馈（加载、空态、错误态缺失） | 🟡 中等 | 体验差，可能被拒 |
| 10 | 页面跳转超过 10 层导致最后跳不动 | 🟡 中等 | 功能 Bug，用户投诉 |
| 11 | 暗黑模式下未适配导致白底白字或黑底黑字 | 🟡 中等 | 体验差 |
| 12 | 使用非官方图标/仿冒品牌图标 | 🟡 中等 | 可能侵权，审核风险 |

- **最佳实践**：设计完成后对照本清单逐项自查，规避审核驳回风险。每次设计 review 过一遍本清单。
