# 六、操作交互规范

## 6.1 手势与基础交互

- **组件/场景**：页面点击、滑动、长按交互
- **适用场景**：全页面所有可交互元素
- **官方标准规范**：所有可交互元素必须有视觉反馈，点击热区充足。使用 `hover-class` 实现点击态。
- **尺寸/参数**：最小点击热区 **≥ 88rpx**
- **交互规则**：
  - 点击：`hover-class` 背景色变化（推荐 `rgba(0,0,0,0.05)`），持续时间 0.1–0.2s
  - 长按：触发 `longpress` 事件（≥ 350ms），用于复制、删除、分享等二级操作
  - 滑动：左右滑动用 swiper，上下滑动用 scroll-view
- **强制要求**：所有可点击控件热区 ≥ 88rpx，避免误触、无响应。
- **禁止设计**：
  - 过小点击区域（< 88rpx）
  - 交互无反馈（点击无任何视觉变化）
  - 手势冲突（如滑动区域嵌套点击元素）
- **最佳实践**：按钮、标签、列表项统一增加点击热区（通过 padding 扩展），提升操作体验。关键按钮增加 16–24rpx 额外热区。

## 6.2 下拉刷新、上拉加载

- **组件/场景**：列表下拉刷新、上拉加载更多
- **适用场景**：信息流、商品列表、数据列表页面
- **官方标准规范**：支持原生下拉刷新（`enablePullDownRefresh`），上拉加载更多需有状态提示。
- **尺寸/参数**：
  - 加载提示文字：**24rpx**
  - 刷新图标：WeUI 原生样式（灰色圆形箭头 + 文字）
  - 下拉触发距离：约 **120rpx**
- **交互规则**：
  - 下拉刷新：页面顶部下拉超过阈值后触发，刷新完成后回弹
  - 上拉加载：距底部约 **100rpx** 时自动触发 `onReachBottom`
  - 加载完成提示"没有更多了"（24rpx，`rgba(0,0,0,0.4)`）
- **状态流转**：
  1. 下拉中 → "下拉刷新"/"释放刷新"
  2. 刷新中 → loading 动画
  3. 刷新完成 → 回弹消失
  4. 加载更多中 → "加载中…"
  5. 全部加载完毕 → "没有更多了"
  6. 加载失败 → "加载失败，点击重试"
- **强制要求**：数据加载完毕必须展示无更多数据提示，禁止无限加载循环。
- **禁止设计**：
  - 无加载完毕提示
  - 重复加载、卡顿加载
  - 上拉加载无节流（用户快速上拉触发多次请求）
- **最佳实践**：短列表（< 20 条）关闭上拉加载，长列表开启加载状态提示。首次加载用骨架屏，刷新用原生下拉。

## 6.3 横屏适配

- **组件/场景**：横屏显示适配
- **适用场景**：视频播放、游戏、AR 场景、部分工具类小程序
- **官方标准规范**：小程序默认竖屏。如需支持横屏，在页面配置中设置 `"pageOrientation": "auto"` 或 `"landscape"`。
- **尺寸/参数**：
  - 横屏时 `rpx` 基准仍为屏幕宽度（与竖屏一致）
  - `wx.getSystemInfoSync().screenWidth` 获取实际屏幕宽度
  - 使用 `match-media` 组件或 `wx.onWindowResize` 监听屏幕旋转
- **强制要求**：横屏时内容布局不可断裂，核心控件仍在安全区内。
- **禁止设计**：横屏布局与竖屏完全相同（不做适配），导致内容拉伸/裁切。
- **最佳实践**：视频类小程序优先支持横屏，一般工具/内容类保持竖屏即可。

## 6.4 iPad 适配

- **组件/场景**：小程序在 iPad 上的显示适配
- **适用场景**：所有小程序（自动兼容）
- **官方标准规范**：小程序在 iPad 上以固定宽高比窗口展示（竖屏约 414px 宽），不可全屏。无需额外适配，但需验证关键页面在 iPad 分辨率下不会异常拉伸。
- **尺寸/参数**：
  - iPad 上小程序窗口宽度约 375–414px（与手机一致的比例）
  - 若开启 `"resizable": true`（桌面端），窗口可自由缩放
- **强制要求**：无。iPad 默认兼容。
- **禁止设计**：依赖特定屏幕宽度的硬编码布局（如 `calc(100vw - 300px)`）。
- **最佳实践**：用 flex 弹性布局 + rpx 单位即可自动适配 iPad 窗口。
