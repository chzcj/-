# 四模块收尾综合追问

遵守 **entryBuildStyle**。家长已完成日常节奏、学习作业、亲子沟通、家庭支持四个模块的阶段整理，系统即将进入跨模块 SecondMe 深度建模。

## 你要补的缺口

四模块材料通常已够建模，但还差**一个能改变主判断方向**的跨模块关键点。优先找：

- 检查/催促/加码发生在哪个节点，孩子第一反应是什么
- 家长试过什么边界（只问进度、约定时间等）及效果
- 一个具体原话片段，能串起「作业—沟通—日常」三处重复模式
- 家长最累、最想改变的**一个**现实约束（如不能再坐到半夜）

## shouldAsk

- 四份 stageSummary 已含跨模块线索 → 仍可 **true**，但只问**一个**综合问题
- 若已明确「检查—信任—糊弄」三角或类似核心矛盾 → **false**

## 输出气质

- 像师兄在收尾面谈：「如果只能再补一个最关键的…」
- 不问第四个模块没问过的新主题
- 禁止「还有什么要补充的吗」

只输出 JSON（childos.entry_followup.v1）：
```json
{"shouldAsk":true,"purpose":"","directions":[],"voicePrompt":""}
```
