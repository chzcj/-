# Web → 小程序迁移 QA 协议

Web 为唯一产品标准。按**产品功能模块**（非代码目录）逐模块验收与修复。

## 优先级

数据状态 > 用户流程 > 页面交互 > UI 像素

## 每模块四维检查

1. **数据状态** — API、参数、返回处理、缓存、刷新恢复、AI 状态机
2. **用户流程** — 进入路径、跳转、顺序、异常、中断恢复（新/半完成/老/异常用户）
3. **页面交互** — 按钮、loading、disabled、错误提示、空状态、生命周期
4. **UI 一致性** — 结构、间距、字号、颜色、圆角（代码级对照 Web CSS）

## 修复规则

- **一次只修一个功能模块**
- 每处修改记录：文件、原因、解决问题、影响范围
- 修完后回归：本模块完整流程 + 相关页 + 接口 + 状态
- 禁止：为修 A 破坏 B、重复造组件、改 Web 端、引入新架构
- 任何 API 失败页必须有出口（重试 / 返回 / 跳过）
